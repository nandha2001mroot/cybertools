About Cloud Metadata

Cloud metadata services provide information about running instances, such as instance identity, user data, and security credentials. Misconfigured metadata endpoints can expose sensitive information.

Common Endpoints:
  • AWS: http://169.254.169.254/latest/meta-data/
  • GCP: http://metadata.google.internal/computeMetadata/v1/
  • Azure: http://169.254.169.254/metadata/instance/
  • OpenStack: http://169.254.169.254/openstack/
Expert Insight
Penetration Testing

In penetration testing, cloud metadata endpoints can reveal sensitive information about cloud instances. Nandha Kumar M's research shows that misconfigured metadata services are a common attack vector.

Cloud Security

Cloud metadata URL checking is essential for identifying potential security vulnerabilities in cloud deployments and preventing SSRF attacks.
